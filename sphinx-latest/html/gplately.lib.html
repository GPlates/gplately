recursive function to find the next point along the polyline

Parameters:

  • line – a list of (lat, lon) in radians

  • distance – the distance to the next point in radians

  • strict – in strict mode, we guarantee the distance. sacrifice the end point if necessary

Accum_length:

parameter to pass down the accumulated length for this recursive function

Returns:

(next point, new line) next point (lat,lon) in radians. None, if no more point new line – the line after taking out this point

gplately.lib.polyline.discretize_polyline(line, distance, strict=False)[source]

travel along a polyline; take a sampling point for each distance; return a list of the sampling points.

Parameters:

  • line – a list of (lat, lon) in radians

  • distance – the distance of the adjacent sampling points in radians

  • strict – in strict mode, we guarantee the distance, sacrifice the end point if necessary. in unstrict mode, we always keep the end point

Returns:

the new line a list of (lat, lon) in radians. If the input line is shorter than distance,

in strict mode, return []; in unstrict mode, return [start_point, end_point]

gplately.lib.rotation.sample_between_two_points(point_a, point_b, distances_to_point_a)[source]

sample points between two points along the great circle according to a list of distances to point a

Parameters:

  • point_a – point a (lat, lon) in radians

  • point_b – point b (lat, lon) in radians

  • distances_to_point_a – a list of distances(in radians) to point a. the distance must less than the distance between point a and point b. invalid distances will produce Nones in the return list

Returns:

a list of sampled points (lat, lon) in radians. The return list has the same length with the list of distances_to_point_a. point_a and point_b will not be included.
